# Test Design: Story 6.4 - Pricing Page with Tier Comparison

**Story:** 6.4  
**Component:** Pricing Page UI  
**QA Engineer:** Quinn (Test Architect)  
**Date:** 2026-01-13  
**Risk Level:** LOW (Primarily presentation layer, minimal business logic)

---

## 1. Test Strategy

### Test Levels
- **Unit Tests** (30%): Pricing calculations, tier helpers
- **Component Tests** (40%): PricingCard components
- **E2E Tests** (30%): Full page navigation and CTA flows

### Focus Areas
- Pricing display accuracy
- Feature comparison correctness
- CTA button functionality
- Mobile responsiveness
- Analytics tracking

---

## 2. Test Scenarios

### Scenario 1: Pricing Display (P0)
**Given-When-Then:**
- **Given** user navigates to /pricing
- **When** page loads
- **Then** all 3 tiers displayed correctly

**Test Cases:**
- TC-6.4-001: Basic tier shows ฿99/month
- TC-6.4-002: Pro tier shows ฿199/month
- TC-6.4-003: VIP tier shows ฿399/month
- TC-6.4-004: "Most Popular" badge on Pro tier
- TC-6.4-005: All prices in Thai Baht (฿)

**Priority:** P0

---

### Scenario 2: Feature Comparison (P1)
**Given-When-Then:**
- **Given** pricing page loaded
- **When** viewing feature lists
- **Then** features match tier configuration

**Test Cases:**
- TC-6.4-010: Basic features list accurate
- TC-6.4-011: Pro features include "All Basic +"
- TC-6.4-012: VIP features include "All Pro +"
- TC-6.4-013: Check marks visible for included features
- TC-6.4-014: Feature count: Basic (6), Pro (5), VIP (6)

**Priority:** P1

---

### Scenario 3: CTA Buttons (P0)
**Given-When-Then:**
- **Given** user on pricing page
- **When** clicking "เริ่มใช้งาน Basic"
- **Then** redirects to /checkout?tier=basic

**Test Cases:**
- TC-6.4-020: Basic CTA → /checkout?tier=basic
- TC-6.4-021: Pro CTA → /checkout?tier=pro
- TC-6.4-022: VIP CTA → /checkout?tier=vip
- TC-6.4-023: Already subscribed → CTA disabled or shows "Current Plan"
- TC-6.4-024: Guest user → redirects to login first

**Priority:** P0

---

### Scenario 4: Mobile Responsiveness (P1)
**Test Cases:**
- TC-6.4-030: Cards stack vertically on mobile
- TC-6.4-031: All text readable on small screens
- TC-6.4-032: CTA buttons accessible
- TC-6.4-033: Feature comparison scrollable
- TC-6.4-034: Responsive breakpoints at 768px, 1024px

**Priority:** P1

---

### Scenario 5: Analytics Tracking (P2)
**Test Cases:**
- TC-6.4-040: pricing_viewed event on page load
- TC-6.4-041: tier_selected event on CTA click
- TC-6.4-042: Time on page tracked
- TC-6.4-043: Scroll depth tracked

**Priority:** P2
