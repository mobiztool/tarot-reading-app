# Test Design: Story 7.8 - Spread Recommendation Engine

**Story:** 7.8  
**Component:** Personalized Spread Recommendations  
**QA Engineer:** Quinn (Test Architect)  
**Date:** 2026-01-13  
**Risk Level:** MEDIUM (Algorithm accuracy)

---

## 1. Test Strategy

### Test Levels
- **Unit Tests** (60%): Recommendation algorithm
- **Integration Tests** (30%): API, personalization
- **E2E Tests** (10%): User sees helpful recommendations

### Focus Areas
- **Algorithm accuracy** (relevant recommendations)
- **Keyword matching** (Thai and English)
- **User history patterns**
- **Tier filtering** (only show accessible spreads)

---

## 2. Test Scenarios

### Scenario 1: Keyword-Based Recommendations (P1)
**Test Cases:**
- TC-7.8-001: Question "love" → love/relationship spreads
- TC-7.8-002: Question "career" → career spreads
- TC-7.8-003: Question "decision" → decision making spread
- TC-7.8-004: Thai keywords detected (ความรัก, การงาน)
- TC-7.8-005: Multiple keywords → combines suggestions

**Priority:** P1

---

### Scenario 2: History-Based Recommendations (P1)
**Test Cases:**
- TC-7.8-010: User with love readings → suggest relationship deep dive
- TC-7.8-011: New user → popular spreads suggested
- TC-7.8-012: Recommends spreads user hasn't tried

**Priority:** P1

---

### Scenario 3: Tier Filtering (P0)
**Test Cases:**
- TC-7.8-020: FREE user → only see free spreads
- TC-7.8-021: PRO user → see free + pro spreads
- TC-7.8-022: VIP user → see all spreads
- TC-7.8-023: Premium spreads show tier badge

**Priority:** P0

---

### Scenario 4: Recommendation Display (P1)
**Test Cases:**
- TC-7.8-030: Shows 3-5 recommendations
- TC-7.8-031: Each includes name, description
- TC-7.8-032: "Why this spread?" explanation
- TC-7.8-033: Click → navigate to spread

**Priority:** P1

---
